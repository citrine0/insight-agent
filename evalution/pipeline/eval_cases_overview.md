# 评测用例总览

> 12 条 Case，覆盖 2 组能力维度，基于电商后台真实导出报表格式

## 数据 Schema

### 主表：店铺经营概况.csv（R01+R02 合并，单平台过滤）

粒度：日 × 品类 × 流量来源

| 中文列名 | 原始字段 | 来源 | 说明 |
|---------|---------|:----:|------|
| 日期 | stat_date | R02 | 合并键 |
| 品类 | category_name | R02 | 合并键 |
| 流量来源 | source_type | R02 | 自然搜索/付费推广/直播/推荐/直接访问 |
| 访客数 | uv | R02 | 来源粒度 UV |
| 浏览量 | pv | R02 | |
| 平均停留时长(秒) | avg_stay_seconds | R02 | |
| 跳失率 | bounce_rate | R02 | |
| 加购人数 | add_cart_cnt | R02 | |
| 支付买家数 | pay_uv | R02 | 来源粒度（与交易表不同维度） |
| 支付转化率 | uv_pay_rate | R02 | pay_uv / uv |
| 支付订单数 | pay_order_cnt | R01 | 日×品类粒度，同一天同品类不同来源行重复 |
| 支付金额 | pay_amount | R01 | 同上 |
| 支付件数 | pay_item_cnt | R01 | 同上 |
| 退款订单数 | refund_order_cnt | R01 | 同上 |
| 退款金额 | refund_amount | R01 | 同上 |
| 客单价 | avg_order_amount | R01 | pay_amount / pay_order_cnt |
| 件单价 | avg_item_price | R01 | pay_amount / pay_item_cnt |

> 注：交易指标（支付订单数、支付金额等）为日×品类粒度，在同一天同一品类的不同流量来源行中是重复值。这是真实平台导出的典型样式。

## 技术说明

- **数据范围**：2025-07-01 至 2025-08-31
- **平台**：淘宝、京东、抖音、微信小店
- **品类**：美妆、食品、电子、服装
- **流量来源**：自然搜索、付费推广、直播、推荐、直接访问
- **所有 Case 的 expected_route 统一为 "complex"**（v9 架构）
- **评测集 JSON 版本**：v3.0，输出文件为 eval_cases_v2.json
